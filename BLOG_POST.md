# AI가 "지금 얼마나 잘하느냐"보다 "얼마나 빠르게 성장하느냐"가 더 중요한 이유

*— NGES를 만든 이유*

---

## 시작은 단순한 의문이었다

AI 모델을 개발하다 보면 자연스럽게 이런 상황이 온다.

새 모듈을 추가했다. 뭔가 달라진 것 같다. 근데 실제로 좋아진 건지 모르겠다.

그래서 벤치마크를 돌린다. MMLU, HumanEval, 혹은 직접 만든 테스트셋. 점수가 나온다.

근데 이 점수, 충분한가?

---

## 기존 벤치마크가 놓치는 것

대부분의 벤치마크는 **스냅샷**이다.

지금 이 순간, 이 모델이 얼마나 잘 푸는가.

이건 중요한 질문이다. 근데 **유일한** 질문이어서는 안 된다.

내가 개발하는 건 한 번 만들고 끝나는 시스템이 아니다. 계속 학습하고, 스스로 수정하고, 새 모듈이 추가되는 구조다. 이런 시스템에서 진짜 중요한 질문은 따로 있다.

> **이 구조가, 시간이 갈수록 실제로 똑똑해지고 있는가?**

이걸 측정하는 도구가 없었다.

---

## 점수보다 기울기

수학 시간에 배운 걸 다시 생각해보자.

위치보다 속도가 더 많은 걸 말해준다.

NGES 60점짜리 시스템 A와 NGES 45점짜리 시스템 B가 있다고 하자.

- 시스템 A: 3개월째 60점 근처에서 정체
- 시스템 B: 매 사이클 +12점씩 성장 중

6개월 후 어느 시스템이 더 앞서 있을까? 당연히 B다.

근데 기존 벤치마크는 이걸 보여주지 못한다.

---

## NGES가 다른 점

NGES는 세 축을 측정한다.

**A축 — 현재 능력 (35%)**
지금 문제를 얼마나 잘 푸는가. 일반적인 벤치마크와 비슷하다.

**B축 — 성장 속도 (45%)**
여기가 핵심이다. 이전 사이클 대비 얼마나 빠르게 나아졌는가.
실패 후 회복하는 속도, 피드백을 흡수하는 능력, Dream 루프 전후의 차이.

**C축 — 구조 효율 (20%)**
같은 자원으로 더 좋은 결과를 내는가. 복잡도가 늘어도 성능이 유지되는가.

B축이 가장 높은 가중치(45%)를 가진다. 의도적인 선택이다.

지금 점수가 낮아도 빠르게 성장하는 구조가, 지금 점수는 높지만 정체된 구조보다 장기적으로 더 가치 있다고 생각하기 때문이다.

---

## 구체적으로 이런 걸 잡아낼 수 있다

메모리 모듈을 새로 붙였다. NGES를 돌렸더니 A축은 거의 그대로인데 B축이 크게 올랐다.

해석: 지금 당장의 정확도는 비슷하지만, 이 모듈이 붙고 나서 다음 사이클에서 더 빠르게 학습하고 있다는 뜻이다. 올바른 방향이다.

Dream 루프를 도입했다. B4(자가 수정 효과)가 올랐다.

해석: 모델이 자기 답을 돌아보고 실제로 수정하는 능력이 생겼다.

새 모듈을 추가했더니 C3(복잡도 안정성)이 떨어졌다.

해석: 구조가 복잡해지면서 성능이 불안정해지고 있다. 모듈 간 인터페이스를 다시 봐야 한다.

이런 것들을 숫자로 잡아내는 게 NGES의 목적이다.

## API 키 없이도 된다

처음 설계할 때 중요하게 생각한 것 중 하나다.

외부 LLM API에 의존하지 않고, 내가 만든 모델만으로 전체 벤치마크를 실행할 수 있어야 한다.

그래서 `RuleBasedJudge`를 만들었다. 정밀도는 LLM Judge보다 낮지만, 인터넷 연결도, API 키도, 비용도 없이 돌아간다.

```python
from nges.judge.rule_judge import RuleBasedJudge

result = NGESRunner(
    model=MyModel(),
    judge=RuleBasedJudge(),  # 외부 의존 없음
    ...
).run(cycle=1)
```

## 오픈소스로 공개한 이유

이걸 비공개로 가져갈 수도 있었다.

근데 NGES의 가치는 "측정 지표"가 아니라 "측정하는 철학"에 있다고 생각했다.

AI 시스템을 단순히 점수로 비교하지 않고, 성장성과 적응성까지 함께 보는 관점. 이게 퍼지는 게 더 의미 있다고 봤다.

평가 체계 자체는 공개하되, 각자가 자신의 시스템에 맞게 확장해 쓸 수 있도록.

---

## 아직 미완성이다

솔직히 말하면 NGES v1.0은 아직 여러 가지가 부족하다.

- 점수 계산식의 가중치가 실험적으로 검증되지 않았다
- 벤치마크 태스크 세트가 충분히 다양하지 않다
- LLM Judge의 일관성이 Judge 모델마다 달라질 수 있다

이런 것들은 계속 개선할 예정이다.

완벽한 도구를 기다리다 아무것도 안 하는 것보다, 불완전하더라도 지금 쓰면서 고쳐나가는 게 낫다고 생각한다.

그게 NGES가 측정하려는 것과도 맞닿아 있다. 성장하는 과정 자체가 중요하다.
