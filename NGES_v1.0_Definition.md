# NGES v1.0 — Nexus Growth Evaluation Standard

## 공식 정의

> **NGES(Nexus Growth Evaluation Standard)** 는 AI 시스템의 "현재 성능"과 "성장 속도"를 통합 측정하는 Nexus AI 전용 평가 프레임입니다.
> 단순 벤치마크 점수를 넘어, 시간에 따라 시스템이 얼마나 빠르고 안정적으로 발전하는지를 수치화합니다.

---

## 1. 평가 구조 개요

NGES는 세 개의 주요 축(Axis)으로 구성됩니다.

| Axis | 명칭 | 설명 | 가중치 |
|------|------|------|--------|
| A | 현재 능력 (Current Capability) | 지금 이 시점의 문제 해결 성능 | 35% |
| B | 성장 속도 (Growth Velocity) | 시간에 따른 개선 속도와 누적 성장 | 45% |
| C | 구조 효율 (Structural Efficiency) | 자원 대비 성능, 모듈 협업 안정성 | 20% |

**NGES 총점 (0 ~ 100)**

```
NGES = (A × 0.35) + (B × 0.45) + (C × 0.20)
```

---

## 2. Axis A — 현재 능력 (Current Capability)

현재 시점에서 시스템이 얼마나 잘 동작하는지를 측정합니다.

### 세부 항목

| 코드 | 항목 | 설명 | 배점 |
|------|------|------|------|
| A1 | 문제 해결 정확도 | 주어진 과제의 정답률 | 30점 |
| A2 | 추론 품질 | 단계적 논리 전개의 깊이와 정확성 | 25점 |
| A3 | 기억 회수 정확성 | 장기/단기 메모리에서 정보를 얼마나 정확히 인출하는가 | 20점 |
| A4 | 응답 일관성 | 동일 문제에 반복 노출 시 응답의 일관성 | 15점 |
| A5 | 장기 목표 유지력 | 멀티턴/장기 세션에서 목표를 끝까지 추적하는 능력 | 10점 |

**Axis A 점수 (0 ~ 100)**

```
A = A1 + A2 + A3 + A4 + A5
```

---

## 3. Axis B — 성장 속도 (Growth Velocity)

NGES의 핵심 차별점입니다. "지금 얼마나 잘하느냐"가 아닌, "얼마나 빠르게 나아지느냐"를 측정합니다.

### 세부 항목

| 코드 | 항목 | 설명 | 배점 |
|------|------|------|------|
| B1 | 개선 속도 | 실패 후 동일 유형 과제에서 오류를 줄이는 속도 | 25점 |
| B2 | 피드백 반응성 | 외부 피드백 수신 후 다음 사이클의 성능 향상 폭 | 20점 |
| B3 | 새 환경 적응력 | 처음 접하는 과제/도메인에 대한 적응 속도 | 20점 |
| B4 | 자가 수정 효과 | Dream/Reflection 등 자가개선 루프 실행 후 성능 상승률 | 20점 |
| B5 | 성장 누적성 | 여러 사이클에 걸쳐 성장이 지속·누적되는지 여부 | 15점 |

**Axis B 점수 (0 ~ 100)**

```
B = B1 + B2 + B3 + B4 + B5
```

### 성장 속도 계산 방법 (B1, B2, B4 공통 참조)

```
개선율(%) = ((T+1 사이클 점수 - T 사이클 점수) / T 사이클 점수) × 100
```

| 개선율 | 점수 |
|--------|------|
| 15% 이상 | 만점 |
| 10~14% | 80% |
| 5~9% | 60% |
| 1~4% | 30% |
| 0% 이하 (퇴행) | 0점 |

---

## 4. Axis C — 구조 효율 (Structural Efficiency)

멀티모듈 구조, 자원 사용, 복잡도 관리를 측정합니다.

### 세부 항목

| 코드 | 항목 | 설명 | 배점 |
|------|------|------|------|
| C1 | 자원 대비 성능 | 동일 연산 자원으로 얼마나 높은 성능을 내는가 | 35점 |
| C2 | 모듈 협업 효율 | 여러 모듈이 충돌 없이 협력하는 정도 | 35점 |
| C3 | 복잡도 안정성 | 모듈/파라미터 수 증가에도 성능이 안정적으로 유지되는가 | 30점 |

**Axis C 점수 (0 ~ 100)**

```
C = C1 + C2 + C3
```

## 5. NGES 등급표

| 등급 | NGES 점수 | 설명 |
|------|-----------|------|
| S | 90 ~ 100 | 탁월한 성능 + 빠른 성장 + 높은 구조 효율 |
| A | 75 ~ 89 | 강한 성장성, 대부분 항목 우수 |
| B | 60 ~ 74 | 평균 이상, 일부 항목 개선 여지 있음 |
| C | 45 ~ 59 | 현재 성능 또는 성장 속도 중 한 축이 약함 |
| D | 30 ~ 44 | 전반적으로 개선 필요 |
| F | 0 ~ 29 | 근본적인 구조/학습 재검토 필요 |

## 6. 측정 주기 및 방법

### 기본 측정 주기

| 유형 | 주기 | 목적 |
|------|------|------|
| 스냅샷 평가 | 매 버전 릴리즈 시 | 버전 간 비교 |
| 연속 추적 평가 | 매 실험 사이클 | 성장 궤적 추적 |
| 비교 평가 | 모듈 변경 전/후 | 구조 변화의 효과 측정 |

### 성장 추적 공식

NGES는 단일 시점 점수가 아닌 **NGES Growth Index (NGI)** 를 함께 제공합니다.

```
NGI = (NGES_현재 - NGES_이전) / 측정_간격(사이클 수)
```

| NGI 값 | 해석 |
|--------|------|
| +2.0 이상/사이클 | 매우 빠른 성장 |
| +1.0 ~ 1.9 | 건강한 성장 |
| +0.1 ~ 0.9 | 완만한 성장 |
| 0.0 | 정체 |
| 음수 | 퇴행 (원인 분석 필요) |

## 7. 활용 시나리오

### 7.1 버전 비교

```
v0.1: NGES 42 (C등급) — 기초 구현 단계
v0.2: NGES 58 (C등급) — 메모리 모듈 추가 후
v0.3: NGES 71 (B등급) — Dream 루프 도입 후
NGI: +14.5/버전 → 빠른 성장 궤적
```

### 7.2 연구 방향 판단

| 시나리오 | 해석 |
|----------|------|
| 점수 높음 + NGI 낮음 | 성능 정체, 구조 재검토 필요 |
| 점수 낮음 + NGI 높음 | 장기적으로 유망한 구조 |
| 점수 높음 + NGI 높음 | 최선의 상태 (S등급 후보) |
| 점수 낮음 + NGI 낮음 | 근본적 방향 수정 필요 |

### 7.3 모듈 수정 효과 측정

```
측정 대상: Dream 로직 수정 전/후
- 수정 전: B4 = 12/20
- 수정 후: B4 = 18/20
- 효과: +30% 향상 → 해당 수정 채택 결정
```

## 8. NGES가 특히 적합한 구조

NGES는 다음과 같은 AI 시스템에 최적화되어 있습니다.

- **멀티모듈 AI**: 여러 전문 모듈이 협력하는 구조
- **자가개선 에이전트**: Dream/Reflection 루프를 가진 시스템
- **장기 실험형 AI**: 단기 벤치마크보다 장기 성장이 중요한 연구용 모델
- **소형 모델 앙상블**: 큰 단일 모델 대신 작은 모델 여럿을 조합하는 구조
